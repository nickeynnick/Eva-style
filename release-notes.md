## Ева-стиль 1.2.0

### Учёт и операции (1.1.1 → 1.2.0)
- **Сброс данных** — при сбросе обнуляются и выплаты администраторам (`adminPaidWages`)
- **Журнал владелицы** — эквайринг за день включает солярий, продажу сертификатов и погашение долгов картой
- **Редактирование визита** — при смене оплаты можно выбрать сертификат и указать клиента для долга
- **Эквайринг долгов** — комиссия банка учитывается при погашении долга картой (как у визитов и сертификатов)
- **Автосохранение резервной копии** — раз в день/неделю в «Документы → Ева-стиль → Backups» (настройка во вкладке «Владелица → Безопасность»)
- **Быстрый поиск** — клиент (имя/телефон), сертификат (номер), должник (ФИО) — иконка лупы в шапке
- **Печать итога смены** — кнопка «Итог дня» на странице «Учёт за день»

### Техническая база
- **Единый store данных** — один модуль с версией схемы (v2) и миграциями; автоперенос из старых ключей localStorage
- **Миграция солярия** — старым сеансам без `minuteRate` проставляется тариф на дату сеанса
- **TypeScript** — `@types/react`, `@types/react-dom`; постепенное ужесточение проверок
- **Автотесты (Vitest)** — расчёт тарифов, эквайринга, сертификатов, зарплаты мастера (`npm run test`)
- **Code-splitting** — вкладки подгружаются отдельными модулями; быстрее старт приложения

### Данные и отчёты
- **Экспорт CSV для Excel** — ведомость мастеров, табель администраторов, выручка по месяцам (раздел «Владелица → Финансы»)
- **Сравнение периодов** — «этот месяц vs прошлый» при выборе периода «Месяц»
- **Импорт резервной копии** — проверка JSON, версия формата, предпросмотр «что восстановится» перед применением
- **Формат бэкапа v2** — поле `backupFormatVersion` в JSON-экспорте

### Развёртывание и обновления
- **«Что нового»** — окно с описанием изменений при первом запуске после обновления
- **Автообновление** — через GitHub Releases (установленная версия)
