# История изменений

Все значимые изменения проекта документируются в этом файле.

Версия в UI / «Что нового» / справке: см. `src/data/appVersion.ts` и `src/data/changelog.ts`.

## [1.2.2] — 2026-07-14

### Добавлено
- **SQLite-хранилище** в Windows-приложении: `Документы/Ева-стиль/Data/eva_style_store.sqlite` (рядом с Backups); localStorage — зеркало
- **Хеширование пароля владелицы** (PBKDF2-SHA-256); старый plaintext мигрирует при успешном входе
- **Автокопии** — расширенные интервалы, ротация последних 10 файлов `eva_style_auto_*.json`, уведомление об успехе или ошибке
- **Удаление долгов** — переключатель в безопасности (как у сертификатов)
- **CrashLogs** — логи сбоев рядом с программой; в режиме разработчика — «Открыть CrashLogs»
- **Тёмная тема** — переключатель справа от Store
- **Обновлённый значок** приложения
- **Фильтр по сотруднику** в истории выплат и начислений
- **Режим разработчика** — история изменений данных, логи, отладка, диагностика
- **Звук при запуске**; подписи в шапке «Сохранить копию» / «Восстановить»

### Изменено
- **Персонализация сводного отчёта** — реально применяет порядок и скрытие блоков
- **История изменений store** пишется всегда (в т.ч. удаление долгов)
- **Справка** — актуальные описания экспорта/импорта, автокопий, SQLite, прав безопасности
- **Синхронизация** справки, окна «Что нового» и CHANGELOG на версию 1.2.2

## [1.2.1] — 2026-07-11

### Исправлено
- **Поля ввода после диалогов** — после alert/confirm/prompt и модальных окон фокус возвращается в форму (Windows/Electron); вместо `window.alert` — встроенные диалоги приложения
- **Печать «Итог дня»** — работает без всплывающих окон и ошибки «about:blank» в Electron
- **Preload-скрипт** — версия приложения передаётся из main-процесса; исправлена ошибка `module not found: ../package.json` при `sandbox: true`
- **P&L и сводки** — эквайринг и безнал за период учитывают солярий, продажу сертификатов и погашение долгов картой (как в журнале)
- **Приветственный экран** — убран «призрачный» overlay, блокировавший клики после анимации

### Добавлено
- **Автотесты** расчёта эквайринга и безнала за период (`dailyFinanceUtils`)

## [1.2.0] — 2026-07-11

### Добавлено
- **Единый store данных** (`eva_style_store`, schema v2) с миграциями из разрозненных ключей localStorage
- **Автотесты** (Vitest): тарифы по дате, эквайринг, сертификаты, зарплата мастера — `npm run test`
- **Code-splitting** вкладок — ленивая загрузка модулей, ускоренный старт
- **Экспорт CSV** для Excel: ведомость мастеров, табель админов, выручка по месяцам
- **Сравнение периодов** «этот месяц vs прошлый» в разделе владелицы (режим «Месяц»)
- **Быстрый поиск** в шапке: клиент, сертификат, должник
- **Печать «Итог дня»** на странице учёта за смену
- **Автосохранение резервной копии** в «Документы → Ева-стиль → Backups» (день/неделя)
- **Проверка импорта** резервной копии с предпросмотром перед восстановлением
- **Окно «Что нового»** при первом запуске после обновления
- **`@types/react` / `@types/react-dom`**; ужесточение TypeScript (`noImplicitAny`, `strictNullChecks`, `strictFunctionTypes`)

### Изменено
- **Редактирование визита** — при смене на «сертификат» или «в долг» доступны выбор сертификата и данные клиента
- **Эквайринг** — единая логика для визитов, солярия, продажи сертификатов и погашения долгов картой
- **Журнал владелицы** — колонка эквайринга учитывает все источники; годовой журнал синхронизирован
- **Формат JSON-бэкапа** — добавлено поле `backupFormatVersion: 2`
- **Миграция солярия** — сеансам без `minuteRate` проставляется тариф на дату операции

### Исправлено
- **Сброс данных** — обнуляются выплаты администраторов (`adminPaidWages`)
- **Автобэкап** — снимок данных актуален на момент сохранения (исправлено устаревшее замыкание)
- **Редактирование визита** — повторное сохранение с полностью использованным сертификатом
- **Предупреждение Vite** о циклических чанках при сборке

## [1.1.1] — 2026-07-11

### Исправлено
- **Тарифы по дате** — системные тарифы (эквайринг, солярий) применяются по правилу с `effectiveDate` ≤ даты операции, как в табеле администраторов
- **Солярий** — тариф ₽/мин сохраняется в записи сеанса; исторические сеансы не пересчитываются по текущему тарифу
- **Сертификаты и долги** — синхронизация баланса сертификата и суммы долга при редактировании и восстановлении визита
- **Эквайринг** — комиссия учитывается при продаже подарочного сертификата картой
- **Резервная копия** — экспорт/импорт дополнен настройками калькулятора, настройками интерфейса и выплатами администраторов
- **Иконки** — исправлены кнопки экспорта и импорта в шапке приложения

## [1.1.0] — 2026-07-11

### Добавлено

- **Прогресс загрузки** обновления отображается на иконке в панели задач и в заголовке окна
- **Ошибки скачивания** теперь показываются пользователю
- **Скрипт публикации** — исправлен выбор файлов (теперь использует точное имя версии вместо glob)
- **Список изменений** в диалоге обновления теперь отображается корректно

## [1.0.5] — 2026-07-11

### Добавлено
- **Автообновление** через GitHub Releases для установленной версии (NSIS)
  - Проверка обновлений при запуске (через 5 секунд)
  - Пункт меню «Справка → Проверить обновления»
- **Портативная версия** — `eva-style-1.0.5-portable.exe` (запуск без установки)
- **Диалог сохранения** резервной копии в десктопной версии (кнопка экспорта в шапке)

### Исправлено
- **Сброс данных** — подтверждение словом «СБРОС» перенесено в интерфейс приложения (в Electron `window.prompt` не работает)
- Сброс данных сразу записывает изменения в `localStorage`
- Версия в меню «О программе» берётся из `package.json` автоматически

### Изменено
- Установщик переименован в `eva-style-setup-1.0.5.exe` для совместимости с автообновлением
- При сборке генерируется `latest.yml` для `electron-updater`

### Примечания
- Автообновление работает только в **установленной** версии, не в portable
- Для автообновления на GitHub Releases должны быть загружены `eva-style-setup-X.X.X.exe` и `latest.yml`
